Project Description

As a database administrator at Cotiviti Technology, I was responsible for managing backups of more than 200+ databases. Although there were cron job with scrips running in place, it was hard to monitor the issue going into each database servers.

To address these challenges, I developed a Centralized Backup Management System using Python, providing a comprehensive running all the jobs from central server, managing all the logs and status and creating easier analytics with a centralised dashboard.

Challenges

  • Inconsistent backup verification across multiple servers
  • Delayed detection of backup failures
  • Inefficient resource allocation causing extended backup windows
  • Limited visibility across the backup ecosystem
  • Complex scheduling requirements for different databases

Solutions

  • Python monitoring engine for continuous tracking of backup status
  • Automated email alerts for immediate notification of failures
  • Multi-threaded execution framework for parallel processing
  • Centralized dashboard for real-time visibility
  • Flexible scheduling system using Python's scheduler library
